### 第三阶段总结

说实话，当时我第二阶段的进度很拖沓，一度产生放弃的想法。但是想着来都来了，还是尝试一下吧。  

第三阶段我选择了 Rust for Linux, 因为这段时间也是在捣鼓 Arch Linux, 也通过第一阶段完成了 Rust 的入门学习，就想着二者结合一下，不过对我来说难度还是有点大了。  

在前期学习的过程中有很多新奇的体验，比如编译 Linux 内核，第一次知道原来内核真的只是个内核，还需要提供一套文件系统才能正常使用等等...  

四个练习虽然是照着老师的视频教程照葫芦画瓢完成的，但看到编写的模块能在控制台打印出信息时还是觉得很神奇，感觉就是像在写用户程序，只不过要用到内核提供的接口。也正是因为这点，不知道需要用到哪些接口，同时时间也紧张，所以我放弃了后续的实习。  

这个过程充满了很多困难，比如刚开始我完全没有“根文件系统”之类的概念，完全不清楚要怎么把内核放到 QEMU 上运行。当然后面老师补充了相关的教程，让我得以继续下去；再就是编辑器提示配置不好，写 Rust 代码的时候非常痛苦，这个在后面尝试实习项目的过程中大概摸索出来了, 不过我也到此为止了。在练习中，我从网卡驱动的示例中了解到了一些 Linux 内核里的一些实现，了解到了一些 unsafe Rust 技巧，也会对我后续对 Linux 的学习有所帮助。  

用 Rust 编写 Linux 驱动是一个新的挑战，但是就我的体验而言，可能标准还不是很规范吧。例如练习使用的 `fujita/linux` 仓库，原作者就自己编写了大量 binding 和对应的 Rust 抽象，这些在 rust-for-linux 官方仓库就没有，如果每个人写驱动都自己实现一堆 binding 和抽象, 想想都会觉得很糟糕吧。另外 Rust 语言可能也不够成熟，一些语言特性还在持续演进当中，项目群里也有学员提到了相关问题。  

总的来说，感谢二位老师的辛勤付出，让我能了解到这个新的领域。虽然过程是曲折的，但是更加激发了我在系统编程方面的兴趣，我也会继续努力学习和成长，希望能够在将来为系统编程领域做出自己的贡献。
